Launch tensorboard:

 bazel build tensorflow/tensorboard:tensorboard
./bazel-bin/tensorflow/tensorboard/tensorboard --logdir=/home/fuego/rosetta-db



Launch the training:

cd /home/fuego/tensorflow/tensorflow/examples/image_retraining

python retrain.py \
  --model_dir=inception \
  --summaries_dir=training_summaries/basic \
  --output_graph=retrained_graph_2018_08_27_newcat.pb \
  --output_labels=retrained_labels_2018_08_27.txt \
  --image_dir=/home/fuego/Desktop/forest_db_cleaned \
  --how_many_training_steps=20000


(To avoid rewriting an old model, consider changing the names of retrained_graph.pb and retrained_labels.txt in the chunk of code above. Note that if you do so, you will have to modify the lines that call these two files in the python script you are running, which should be either: par_segmenting_server.py, label_image.py or label_image_reconstructed.py. See 'How-to-run' and 'How-to-test' for more information.)

**IMPORTANT NOTE for running par_segmenting_server.py: Retraining the network will result in retrained_graph.pb and retrained_labels.txt being created (or overwritten) inside /home/fuego/tensorflow/tensorflow/examples/image_retraining. 
If you want to use your new network in the form of par_segmenting_server.py, copy the two files into the folder: /home/fuego/Documents/fuego-server/fuego_server
